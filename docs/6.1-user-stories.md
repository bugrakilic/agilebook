# User Stories

User Stories are short, simple descriptions of a feature told from the perspective of the end user.

## Purpose

- Capture requirements in a user-centric way  
- Facilitate collaboration between stakeholders and the team  
- Provide a basis for estimation and planning  

## Format

A common template is:  
*As a [type of user], I want [some goal] so that [some reason].*

Example:  
*As a registered user, I want to reset my password so that I can regain access to my account.*

## Characteristics

- Focus on value to the user  
- Small and testable  
- Negotiable and flexible  

## Benefits

- Encourages conversation around requirements  
- Helps break down complex features  
- Supports iterative development  

## Acceptance Criteria

Acceptance Criteria define the conditions that a User Story must satisfy to be considered complete and working from the user's perspective. They provide clarity on the expected behavior and help the Development Team understand what needs to be built.

Typically, Acceptance Criteria are written as a list of statements, often in a "Given-When-Then" format (also known as Gherkin syntax) to specify a scenario:
*   **Given** [a context]
*   **When** [an action is performed]
*   **Then** [an observable outcome occurs]

Example for "As a registered user, I want to reset my password so that I can regain access to my account.":
*   **Given** I am on the password reset page
*   **And** I enter my registered email address
*   **When** I click "Send Reset Link"
*   **Then** I should receive an email with a password reset link
*   **And** I should see a confirmation message on the page

## Story Mapping

Story Mapping is a collaborative technique used to visualize the entire user journey of a product and organize user stories into a coherent narrative. It helps teams maintain a holistic view of the product, prioritize features, and plan releases effectively.

Key aspects of Story Mapping:
*   **User Activities:** The main steps a user takes to achieve a goal.
*   **User Tasks:** The specific actions within each activity.
*   **User Stories:** Detailed descriptions of features that support the tasks.
*   **Narrative Flow:** Arranging stories horizontally to represent the user's journey over time, and vertically by priority.
*   **Release Planning:** Drawing "lines" across the map to define minimal viable products (MVPs) or subsequent releases.

Story Mapping helps teams discover gaps in their understanding, build a shared vision, and ensure that they are delivering value in a user-centric sequence.
